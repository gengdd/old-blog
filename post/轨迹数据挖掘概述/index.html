轨迹数据处理

噪声滤波

  1. 均值(中位数)滤波
    • 在处理极端误差时，中值滤波比均值滤波具有更强的鲁棒性
    • 当处理多个连续噪声点时，需要更大尺寸的滑动窗口
    • 当轨迹采样率很低时，均值滤波器和中值滤波器不再是很好的选择
  2. 卡尔曼(粒子)滤波
    • 卡尔曼滤波器通过假设线性模型加上高斯噪声来提高效率，而粒子滤波器则放宽了这些假设
    • 如果轨迹上的第一个点是有噪声的，两个滤波器的有效性显著下降
  3. 基于启发式的异常点检测
    • 通过使用离群点检测算法直接从轨迹中去除噪声点
    • 基于距离的异常检测

驻留点检测

  1. 单点位置
    • 这种情况是非常罕见的，因为用户的定位设备通常产生不同的读数，即使在同一位置
  2. 人们走动，或保持静止但定位读数会四处移动

文献：

[1] Mining user similarity based on location history

轨迹压缩

  1. 离线压缩(即批处理模式):通过从原始轨迹中剔除一些误差可以忽略的点来生成近似轨迹
    • Douglas-Peucke($N^2$)
    • ($N\log N$)(Speeding up the Douglas-Peucker line simplification algorithm)
    • Bellman’s algorithm($N^3$)
  2. 在线压缩:即在物体运动时立即压缩轨迹
    • 基于窗口的算法
      • Sliding Window Algorithm
      • Open Window Algorithm
    • 基于移动物体的速度和方向

  1. 轨迹语义压缩
    • Trajectory Simplification (TS) algorithm
  2. 考虑受交通网络约束的轨迹压缩
    • PRESS(PRESS: A novel framework of trajectory compression in roadnetworks)

两个距离度量度量压缩的误差：

  1. 垂直欧式距离
  2. 时间同步欧式距离

轨迹分割

  1. 基于时间间隔
  2. 基于轨迹的形状
    • Minimal Description Language (MDL)
  3. 基于轨迹中点的语义意义
    • 将轨迹划分为不同运输方式的分段

地图匹配

根据所使用的附加信息，地图匹配算法可以分为四类：

  1. 几何
    • 几何地图匹配算法考虑道路网络中单个链接的形状，例如，匹配GPS点到最近的道路
  2. 拓扑
    • 利用Frechet距离测量GPS序列与候选道路序列之间的拟合(On map-matching vehicle tracking data)
  3. 概率
    • 为了处理噪声和低采样率的轨迹，概率算法考虑通过路网的多种可能路径，以找到最佳路径
  4. 其他
    • Map-matching for low-sampling-rate GPS trajectories
    • Hidden Markov map matching through noise and sparseness

根据所考虑的采样点范围，地图匹配算法可以分为两类：

  1. 局部/增量
    • 运行非常有效，经常采用在线应用程序
    • 当轨迹采样率较低时，匹配精度会降低
  2. 全局
    • 将整个轨迹与路网匹配
    • 比局部方法更精确，但效率较低，通常用于离线任务

轨迹数据管理

轨迹索引与检索

文献：

[1] Trajectory indexing and retrieval

  1. K近邻(点/轨迹)查询
    • 查询点的数量通常非常小，并且在应用程序中可能彼此很远
    • 检索穿过特定路径的车辆轨迹
      • 将路网中的路径视为轨迹，利用KNN轨迹查询来检测与路径接近的轨迹
      • 利用地图匹配算法将轨迹转换为一系列路段，建立一些索引结构来管理路径和通过它们的轨迹之间的关系
        • suffix-tree-based索引结构
        • 由于索引的规模随着轨迹数量的增加而迅速增长，只适合管理近期生成的轨迹

  1. 范围查询
    • 基于轨迹构建3D-Rtree(ST-R-tree和TB-tree)
    • 随着时间的推移，不同的3D box之间的重叠仍然在不断增加
    • 将一个时间段划分为多个时间间隔，为每个时间间隔产生的轨迹建立一个单独的空间索引，如R树
      • Rt-Tree
      • HR-Tree
      • H+R-Tree
    • 将地理空间划分为网格，然后为每个网格中的轨迹建立时序索引
      • CSE-Tree

轨迹距离/相似性

  1. 点和轨迹的距离
    • $D(q,A)=\min _{p\in A}D(p,q)$
    • $D(Q,A)=\sum _{q\in Q}e^{D(q,A)}$, $S(Q,A)=\sum _{q\in Q}e^{-D(q,A)}$
  2. 轨迹和轨迹的距离
    • Closest-pair distance
      • $CPD(A,B)=\min _{p\in A, p^{'}\in B}D(p,p^{'})$
    • Sum-of-pairs distance
      • $SPD(A,B)=\sum _{i=1}^nD(p_i,p_i^{'})$
    • Dynamic Time Wrapping (DTW) distance
    • Longest Common Subsequence(LCSS) LCSS-based distance
    • Edit Distance on Real Sequence(EDR) distance
    • K-BCT
  3. 轨迹段的距离
    • 基于轨迹段的最小边界矩形(MBR)
    • Trajectory-Hausdorff Distance

序列模式

Sequential Patterns是指一定数量的移动物体在相似的时间间隔内沿着公共序列的位置移动

共同序列$l_1\rightarrow l_2\rightarrow l_4$

为了从轨迹中检测序列模式，我们首先需要定义序列中的一个(公共)位置。然而，在许多GPS轨迹中，每个点都具有一对GPS坐标的特征，这些坐标不会在每个模式实例中完全重复

  1. Sequential Pattern Mining in a Free Space
    • Line-Simplification-Based Methods
    • Clustering-Based Methods
      • 检测驻留点作为聚类点
  2. Sequential Pattern Mining in a Road Network

周期模式

两阶段检测方法：

  1. 使用KDE等基于密度的聚类算法来检测运动目标经常访问的参考点
  2. 通过使用分层聚类算法总结部分移动序列的周期性行为

轨迹分类

交通方式分类：驾车、自行车、步行、公交车

三个主要步骤：

  1. 利用分割方法将轨迹分割成若干段
  2. 从每个片段中提取特征
  3. 构建模型以对每个段（或点）进行分类

由于轨迹本质上是一个序列，我们可以利用现有的序列推理模型，如动态贝叶斯网络(DBN)、HMM和条件随机场(CRF)，它们合并了来自局部点(或段)和相邻点(或段)之间的序列模式的信息
